# Proposal: Verify Merchant Dashboard

## Why
We need to verify the complex merchant dashboard functions correctly across all views, including shell navigation, metrics aggregation, staff management inside the dashboard, tips sub-tabs, reviews list filters, physical device registration, and analytics tables.

## What Changes
We will run both unit tests and E2E tests for the merchant dashboard.

## Capabilities
- `verify-merchant-dashboard`: Verifies layout routing, metric boxes, staff rosters/actions, savings calculations, reviews filtering, transactions reporting, devices adding/deleting, and analytics graphs.

## Impact
No direct application code modifications are required for this verification change.
